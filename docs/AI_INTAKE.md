# ZenFlow — AI Intake System

> The adaptive intake questionnaire uses LangChain + Ollama (local LLM) to conduct
> a short, medically-informed conversation before each patient session.

---

## Overview

Before an acupuncture session, the bot asks the patient up to 5 adaptive questions. Each question is generated by a local LLM (`gemma3:latest` via Ollama) based on the patient's previous answers. After the 5th answer, the LLM generates:

1. A **clinical summary** — 4–5 bullet points for the acupuncturist
2. A **TCM diagnosis** — structured JSON with pattern, treatment principles, suggested points, and lifestyle recommendations

Both outputs are saved to SQLite. The clinical summary is also shown to the patient as a booking confirmation.

---

## Components

### LLM Singleton (`ai_intake.py`)

```python
_LLM = ChatOllama(model=OLLAMA_MODEL, base_url=OLLAMA_HOST)
```

- Created once at module import
- `OLLAMA_MODEL` from `.env` (default: `gemma3:latest`)
- `OLLAMA_HOST` from `.env` (default: `http://localhost:11434`)
- `validate_model_on_init=False` (default) — no connection on creation, only on first `ainvoke()`
- All LLM calls use `asyncio.wait_for(..., timeout=100)` — 100-second timeout per call

### Ollama Health Check

```python
_check_ollama_health()
```

Runs at module import. Attempts `GET {OLLAMA_HOST}/api/tags` with a 3-second timeout. Logs a clear warning with `ollama serve` instructions if unreachable. Does not abort startup — the bot starts regardless, using fallback questions if Ollama is unavailable.

## Conversation History Backend

History is stored in **Redis** via `langchain-redis` `RedisChatMessageHistory`:

```python
RedisChatMessageHistory(
    session_id="zenflow:intake:{patient_id}:{therapist_id}",
    url=REDIS_URL,
    ttl=1800,   # 30 minutes
)
```

**Why Redis instead of in-memory:**
- Intake history survives bot restarts
- Web process can theoretically inspect history if needed
- Aligns with the overall architecture of Redis as the shared messaging layer

**In-process object cache** (`_history_cache: dict[int, RedisChatMessageHistory]`): The `RedisChatMessageHistory` object is cached in a Python dict to avoid issuing a Redis `LRANGE` on every LLM call. The object is dropped from the dict on `clear_intake()`.

## Context Window Management (Rolling Summary)

To prevent the LLM context from growing unbounded across 5+ answers, a compression mechanism is applied:

```
_BUFFER_KEEP = 4    messages to keep in live history
_BUFFER_MAX  = 6    trigger compression when history exceeds this
```

**Compression flow (`_maybe_compress`):**
```
if len(history.messages) > 6:
    to_compress = messages[:len-4]       # older messages
    recent      = messages[len-4:]       # keep most recent 4

    if existing _rolling_summaries[user_id]:
        prompt = "Previous summary: {existing}\nNew exchanges: {to_compress}"
    else:
        prompt = BUFFER_SUMMARIZE_PROMPT + transcript

    LLM call (30s timeout) → compressed 2–3 sentence summary
    _rolling_summaries[user_id] = summary

    history.clear()
    history.re-add(recent messages only)
```

**Context construction for each LLM call:**
```python
if _rolling_summaries.get(user_id):
    context = [
        SystemMessage(SYSTEM_PROMPT),
        SystemMessage(f"[Conversation so far: {summary}]"),
        *history.messages[-4:],           # last 4 messages
    ]
else:
    context = [SystemMessage(SYSTEM_PROMPT)] + history.messages[-3:]
```

## System Prompts

### `SYSTEM_PROMPT` — intake conversation rules

Key directives given to the LLM for every question-generation call:
- Ask ONE question per message, never more
- Adapt every question to the patient's previous answer
- Keep questions short and conversational, no medical jargon
- Accept short answers ("yes", "no", "כן", "לא") and follow up meaningfully
- Conclude in 5 exchanges, never exceed 7
- Ask questions in English; if patient replies in Hebrew, continue in English
- Do not include greetings or pleasantries

### `SUMMARY_INSTRUCTION` — clinical summary

Appended as the final message in the `generate_summary()` call:
- Write a concise clinical summary for the acupuncturist
- Include: chief complaint, key symptoms, TCM patterns if apparent, suggested focus areas
- 4–5 bullet points in English

### `TCM_DIAGNOSIS_PROMPT` — structured JSON diagnosis

Appended as the final message in `generate_tcm_diagnosis()`. Requests a strictly-formatted JSON object with:
- `tcm_pattern` — primary TCM syndrome
- `treatment_principles` — e.g. "Move Liver Qi, nourish Blood"
- `diagnosis_certainty` — integer 0–100 (AI confidence)
- `suggested_points` — list of objects `{"code": "ST36", "rationale": "..."}`
- `recommendations` — object with `diet`, `sleep`, `exercise`, `stress` keys

The prompt explicitly requests: "Return ONLY valid JSON with no additional text or markdown."

## Public API Functions

### `initialize_intake(user_id, opening_question)`

```
Called by: start_intake() in schedule.py

1. _history_cache.pop(user_id)          — drop stale in-process cache
2. _rolling_summaries.pop(user_id)
3. _get_history(user_id).clear()        — Redis DEL zenflow:intake:{uid}:{tid}
4. hist.add_ai_message(opening_question) — Redis RPUSH + EXPIRE 1800
```

Opening question is always the static `OPENING_QUESTION` constant:
`"What's the main issue or discomfort bringing you in today?"`

---

### `get_next_question(user_id, user_answer) → str`

```
Called by: handle_intake_answer() for answers 1–4

1. hist.add_user_message(user_answer)   — Redis RPUSH
2. await _maybe_compress(user_id)       — compress if > 6 messages
3. Build context (system + rolling summary + recent messages)
4. await asyncio.wait_for(_LLM.ainvoke(context), timeout=100)
   → on success: hist.add_ai_message(question), return question
   → on TimeoutError: fallback
   → on any Exception: fallback

Fallback: FALLBACK_QUESTIONS[min(answered_count, 4)]
```

---

### `generate_summary(user_id, final_answer) → str`

```
Called by: handle_intake_answer() on 5th answer

1. hist.add_user_message(final_answer)
2. Build context including rolling summary if exists
3. Append SUMMARY_INSTRUCTION as final HumanMessage
4. await asyncio.wait_for(_LLM.ainvoke(messages), timeout=100)
   → on success: return summary string
   → on any error: return "Intake completed — see conversation history for details."
```

---

### `generate_tcm_diagnosis(user_id, clinical_summary) → dict`

```
Called by: handle_intake_answer() after generate_summary()

1. Build context: system + rolling summary + full history + summary + TCM_DIAGNOSIS_PROMPT
2. await asyncio.wait_for(_LLM.ainvoke(context_parts), timeout=100)
3. Strip markdown code fences (```json ... ```) if present
4. Extract JSON object using regex: re.search(r"\{[\s\S]*\}", raw)
5. json.loads(extracted)
6. Normalise suggested_points: accept list of dicts OR list of strings
7. Validate and return typed dict

On any error (timeout, JSON parse error, other):
    return fallback = {
        "tcm_pattern": "",
        "treatment_principles": "",
        "diagnosis_certainty": 0,
        "suggested_points": [],
        "recommendations": {"diet":"", "sleep":"", "exercise":"", "stress":""}
    }
```

---

### `get_history_dicts(user_id) → list[dict]`

```
Called by: handle_intake_answer() to get history for save_appointment()

Converts LangChain message objects to plain dicts:
    HumanMessage → {"role": "user",      "content": "..."}
    AIMessage    → {"role": "assistant", "content": "..."}

Returns the full history as stored in Redis at the time of call.
Written to SQLite intake_sessions.history_json.
```

---

### `clear_intake(user_id)`

```
Called by: handle_intake_answer() and skip_intake() after appointment saved

1. _get_history(user_id).clear()   — Redis DEL zenflow:intake:{uid}:{tid}
2. _history_cache.pop(user_id)
3. _rolling_summaries.pop(user_id)
```

## Fallback Questions

Used when Ollama is unavailable or a call times out:

```python
FALLBACK_QUESTIONS = [
    "What's the main issue or discomfort bringing you in today?",
    "How long have you been experiencing this?",
    "Does it tend to get worse at a particular time of day?",
    "How would you rate the intensity on a scale of 1 to 10?",
    "Have you had any treatment for this before?",
]
```

The fallback question is selected by `FALLBACK_QUESTIONS[min(answered_count, 4)]`, ensuring the index never goes out of bounds.

The appointment is still saved correctly when fallback questions are used. The clinical summary falls back to the generic string, and the TCM diagnosis dict has all blank fields.

## Complete Intake Lifecycle

```
start_intake()
    └─ initialize_intake()
       └─ clear Redis history key
       └─ write OPENING_QUESTION to Redis

Patient answers question 1
    └─ handle_intake_answer() → intake_count = 1
    └─ get_next_question()
       └─ add answer to Redis history
       └─ maybe compress (no, only 2 messages)
       └─ LLM call → question 2

... (repeat for answers 2, 3, 4) ...

Patient answers question 5
    └─ handle_intake_answer() → intake_count = 5
    └─ "Thank you! Saving your details... ⏳"
    └─ generate_summary()
       └─ add final answer to Redis history
       └─ LLM call → clinical summary (4–5 bullets)
    └─ get_history_dicts()
       └─ export Redis history as plain dicts
    └─ book_slot()
       └─ invalidate Redis availability/slots caches
       └─ update calendar
    └─ save_appointment()
       └─ SQLite BEGIN
          INSERT appointments (summary = clinical_summary)
          INSERT intake_sessions (history_json = exported dicts)
          COMMIT
       └─ Redis DEL zenflow:apts:all
    └─ generate_tcm_diagnosis()
       └─ LLM call → structured TCM JSON
    └─ save_treatment_notes()
       └─ SQLite UPSERT treatment_notes
    └─ clear_intake()
       └─ Redis DEL intake key
       └─ drop in-process cache entries
    └─ context.user_data.clear()
    └─ send confirmation to patient (with summary snippet)
```

## Ollama Requirements

| Requirement | Value |
|---|---|
| Install | https://ollama.com |
| Start | `ollama serve` (done automatically by `startup/launch.py`) |
| Default model | `gemma3:latest` (= `gemma3:4b`, 3.3 GB) |
| Pull model | `ollama pull gemma3:latest` |
| Change model | Set `OLLAMA_MODEL` in `.env` |
| Timeout per call | 100 seconds |
| Compression timeout | 30 seconds |

curl http://localhost:11434/api/tags
